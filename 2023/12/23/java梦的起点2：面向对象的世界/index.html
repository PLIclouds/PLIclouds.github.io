Java梦的起点2：面向对象的世界

发表于 2023-12-23 更新于 2024-01-26 分类于 后端 ， Java 阅读次数： 本文字数： 1.6k

面向对象杂谈

面向对象，这四个字可真够有意思的。引用书中的一句话：

类实质上就是封装对象属性和行为的载体，而对象则是从类抽象出来的一个实例。

面向对象认为研究事物只需要研究清楚它的属性和行为就是将一个事物研究透了。怎么说呢，看起来有点机械论的样子，忽略事物发展规律了。但总之，现在大部分程序设计语言都是这么设计的。将事物抽象成属性和行为的载体，倒也不失一种简化方法。

面向对象的三大特质：封装性，继承性，多态性。

封装性强调了外部的不可干扰性。类只提供一些端口，外部只能通过这些端口操作类。至于类的内部结构功能，外部无法知晓也无法修改。

继承性和多态性都是在说类与类之间的关联。继承性指的是，子类可以继承父类的方法和属性。多态性指的是同一操作可以作用于不同对象。

继承性以子类为主体，说明了子类可以使用父类的方法，拥有父类的属性。而多态性则是将父类的引用指向子类的对象。这一段或许目前看起来难以理解，下一章我们重点研究Java在这两种性质上的实现。

这篇文章详细讲了四个修饰符及其使用规则。

非访问修饰符有final,abstract和static。大致作用为：
* static：静态的，非实例的，类的。可以修饰：内部类，成员
* final：最终的，不可改变的。可以修饰：类，成员，局部变量
* abstract：抽象的，不可实例化的。可以修饰：类，成员

后续会更加详细讲这三个修饰符。

总结

本节我们研究Java里“类”这个概念。

类有两类修饰符，分别是四个访问修饰符public,protect,(default),private和两个非访问修饰符abstract,final，前者控制访问权限，后者控制该类在类关系中的性质。

类中可以定义变量和方法，这些变量和方法同样可以由四个访问修饰符和两个非访问修饰符修饰。但他们多一种非访问修饰符static。它可以让变量和方法转换为类变量和类方法。

类中有三种种特殊方法：构造方法可以用来完成对象的创建，重载方法可以完成相同函数名不同参数的函数的创建，主方法是程序的入口（可选）

类中还有一种特殊的用法：静态代码块。这个代码块会在类被第一次创建时执行一次。
