SQL是什么？

关系型数据库是一个结构化的数据库，创建在关系模型（二维表格模型）基础上，一般面向于记录。

SQL(Structured Query Language)，结构化查询语言，就是关系型数据库的操作语言。二维表格，也被叫做表(table)，就是它的操作对象。这样的系统我们称其为关系型数据库管理系统（RDBMS）。

一个数据库中包括多个表，每个表由一个名字标识（即它的名字）和表包（即它本身）组成。表包每一行称为一个记录，通常记录着一个物体的各种属性。而每一列则记录着各种物体的同一属性。由于以列研究数据更加有意义，SQL大部分操作都是以列为单位进行操作。

SQL分为四类，它们分别是：
DDL（Data Definition Language）：数据定义语言，用来定义数据库对象：库、表、列等；
DML（Data Manipulation Language）：数据操作语言，用来定义数据库记录（数据）；
DCL（Data Control Language）：数据控制语言，用来定义访问权限和安全级别；
DQL（Data Query Language）：数据查询语言，用来查询记录（数据）

常用的是DML和DQL。

数据储存格式

SQL规定了丰富的格式存储格式，这里放一些常用的：
* int：整型
* double：浮点型，例如 double(5,2)表示最多 5 位，其中必须有 2 位小数，即最大值为 999.99；
* decimal：泛型型，在表单线方面使用该类型，因为不会出现精度缺失问题；
* char：固定长度字符串类型；(当输入的字符不够长度时会补空格)
* varchar：固定长度字符串类型；
* text：字符串类型；
* blob：字节（bool）类型；
* date：日期类型，格式为：yyyy-MM-dd；
* time：时间类型，格式为：hh:mm:ss

DQL（Data Query Language）

选择数据有很多关键字，但是其实和英语逻辑一模一样。这些关键字都是顺序排列的，念出来是什么效果就是什么效果了（基本遵循“先念先处理”的逻辑）：
* select 列名 ----> 要查询的列名称
* from 表名 ----> 要查询的表名称
* where 条件 ----> 行条件
* group by 分组列 ----> 对结果分组
* having 分组条件 ----> 分组后的行条件
* order by 排序列 ----> 对结果分组
* limit 起始行, 行数 ----> 结果限定

当然有时候还需要对字符串模糊查询，这里要用到LIKE和通配符%和_。前者表示任意长度（包括0）字符，后者表示正好一个字符。

还有时候我们需要用聚合函数做一些统计上的选择，这些函数经常搭配group by 分组列来做统计分析。

多表查询

有时候需要两张三张表拼接在一起查询，这个时候要用到连接查询（多表查询）。我喜欢连接查询（多表查询）——MySQL的思路，把这两张表看作一个数据的集合，然后讨论各种情况。

要连接，那肯定要有相同的列名，比如两个表中都有“名字”这一项。但一个表里有的名字并不能保证另一个表里也有，因此就产生了两个集合，也就产生了内连接，左连接，右连接三种连接方式。

多态

在上一个小节我们讨论了向上转型的安全性，这其实就是多态的基础。我们知道，类的成员方法只能作用于本类，同一个方法如何作用于不同对象？这就要用到上面说到的对象类型转换中的向上转型了。通过向上转型将子类对象当作一个父类对象来解释，就可以将子类对象运用于父类的成员方法了。

例如，如果我需要创建一个Shape类，里面有一个成员函数是draw()，图像类下继承有Square，Circular类。如果不使用多态性质，我需要给每一个子类都创建一个画图成员方法，但是有了多态性质，我只需要把子类解释成一个Shape，在Shape类中重写draw()函数即可，我不需要关注具体的形状类型，我只需要画这个图形。

抽象类

上一个小节说到在Shape类里创建画图函数，但这会产生一个问题：Shape类对象本身是不能调用这个函数的，因为一个抽象概念的图像是画不出来的。于是我们需要引入抽象类的概念。

Java中使用abstract关键字定义抽象类，该类里抽象的成员方法被称为抽象方法。抽象类唯一的意义就是被继承，抽象方法本身也没有方法体，抽象类不可被实例化为对象。

对于抽象类的继承类，被继承后必须实现里面的所有抽象方法。

总结

本章主要研究Java对于类的关系的分析。对于除内部类以外的类，它们之间可能有继承的“父子关系”，子类继承父类的性质，并可以对其进行重写。

为了达到对不同类型对象调用同一函数的目的，可以对子类对象向上转型来实现多态。有时父类或者父类中的方法没有意义，则可以借助abstract来解决这一问题。同时，接口的存在让Java中更方便地在子类中配置方法。

Java梦的起点2：面向对象的世界

发表于 2023-12-23 更新于 2024-01-26 分类于 后端 ， Java 本文字数： 1.6k

面向对象杂谈

面向对象，这四个字可真够有意思的。引用书中的一句话：

类实质上就是封装对象属性和行为的载体，而对象则是从类抽象出来的一个实例。

面向对象认为研究事物只需要研究清楚它的属性和行为就是将一个事物研究透了。怎么说呢，看起来有点机械论的样子，忽略事物发展规律了。但总之，现在大部分程序设计语言都是这么设计的。将事物抽象成属性和行为的载体，倒也不失一种简化方法。

面向对象的三大特质：封装性，继承性，多态性。

封装性强调了外部的不可干扰性。类只提供一些端口，外部只能通过这些端口操作类。至于类的内部结构功能，外部无法知晓也无法修改。

继承性和多态性都是在说类与类之间的关联。继承性指的是，子类可以继承父类的方法和属性。多态性指的是同一操作可以作用于不同对象。

继承性以子类为主体，说明了子类可以使用父类的方法，拥有父类的属性。而多态性则是将父类的引用指向子类的对象。这一段或许目前看起来难以理解，下一章我们重点研究Java在这两种性质上的实现。

这篇文章详细讲了四个修饰符及其使用规则。

* abstract：抽象的，不可实例化的。可以修饰：类，成员

后续会更加详细讲这三个修饰符。

总结

本节我们研究Java里“类”这个概念。

类有两类修饰符，分别是四个访问修饰符public,protect,(default),private和两个非访问修饰符abstract,final，前者控制访问权限，后者控制该类在类关系中的性质。

类中可以定义变量和方法，这些变量和方法同样可以由四个访问修饰符和两个非访问修饰符修饰。但他们多一种非访问修饰符static。它可以让变量和方法转换为类变量和类方法。

类中有三种种特殊方法：构造方法可以用来完成对象的创建，重载方法可以完成相同函数名不同参数的函数的创建，主方法是程序的入口（可选）

类中还有一种特殊的用法：静态代码块。这个代码块会在类被第一次创建时执行一次。
