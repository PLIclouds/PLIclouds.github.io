算法入门2-6：搜索优化

发表于 2023-12-07 更新于 2023-12-09 分类于 算法入门 ， 基础算法和基本思想 本文字数： 1.5k

针对代码的优化应该回归方法，针对方法的优化应该回归思想。我们前文已经分析过，很多情况枚举可能答案是时间复杂度产生的主要原因，那有没有可能，缩小可能的答案集合，就可以达到比较好的效果呢？这就是搜索算法的朴素优化方案。它不存在一个通用的办法，而是考虑解决问题的一个方向，这种做法也被称为剪枝。另外，记忆化搜索和优化搜索路径也是两个有效的优化搜索的方法。

搜索

为了更加彻底地了解搜索这一方法，我们必须提前引入图论里的一些概念。

图 (graph) 是一个二元组 \(G=(V(G), E(G))\)。其中\(V(G)\)是非空集，称为点集 (vertex set) ，其中的每个元素，我们称其为顶点 (vertex)或节点 (node)，简称点 ；\(E(G)\)为各结点之间边的集合，称为边集 (edge set) 。

这个概念总的来说，图就是由点和边组成的。边分为有向边和无向边。

至于“树”的概念嘛，从图形上理解会比从严谨图论概念上理解更加容易。

这个图有一点不太合理，“树”这种数据结构一般不会用有向边来表示，大部分的树都是无向边构成的。

可以发现，“树”这种数据结构，最大的特点就是从一个点发出延申开来，我们把最顶端的点\(A\)称为根节点。扩展到最外端的节点\(K\)，节点\(O\)，节点\(P\)，节点\(J\)和节点\(L\)称作叶节点或者叶子节点。这很像现实生活中的树，从一个点出发，向下生长（当然现实中树是向上生长的），最下面的就是一些叶子了。

我们定义了一些连接关系。以上图的树为例，例如我们称节点\(B\)是节点\(D\)的父节点；相反地，我们称节点\(D\)是节点\(B\)的子节点。我们还称节点\(A\)和节点\(B\)是都是节点\(D\)的祖先。

我们还定义了节点的深度。一个节点和根节点之间相隔的边的数量称为节点的深度，例如节点\(A\)的深度为0（因为自己就是根节点），\(B\)和\(C\)的深度为1，\(O\)的深度为4。

如果有仔细观察的同学一定会发现，如果不指定根节点的位置，树上的任何一个点都可以当作根节点。当然暂时来看这不重要，说明这一点只是为了更好地理解上面的定义。一个节点和另一个节点的关系和该节点的深度，只有在确定了根节点后才有定论。

搜索算法和树的关系

从上面介绍深度开始，如果你已经学明白了之前的内容，你一定已经明白这个关系。其实整个搜索过程就可以等效为遍历这棵树的过程。根节点是搜索的开始，叶子节点就是搜索的结束。深度优先搜索优先向更深的方向搜索，以上面的树为例，遍历顺序为：

\(A\to B\to D\to I\to K\to E\to J\to F\to L\to C\to G\to M\to O\to H\to N\to P\)

可能这样还是有点抽象，我们把这题换成一道算法题：

一只蜗牛要爬到树上去吃树叶，输入是这颗树，怎样爬才能最快吃到叶子？

按照上面的方法搜索，从根节点开始，搜到叶子节点后更新最小深度，直到遍历结束即可完成答案。

所有的搜索都可以转化为这样的一颗树，这是思考搜索问题的一个途径。

更好的搜索策略

或许你在思考上面的问题时发现了一个问题，为什么要深度优先？这个策略看起来不是很聪明的样子。能想到这，说明还是很有质疑精神并且善于发现的。确实，这样的搜索策略浪费了资源，把整个树遍历了一遍。与深度优先搜索DFS对应，还有一种搜索方法叫做广度优先搜索，这是将同一个深度的所有节点搜索完一遍后再搜索下一个深度的搜索策略。依照这个策略，我们只需要在搜索时判断是否是叶子节点，如果是即可立即输出答案，因为按照这个策略，我们保证了这是深度最小的叶子节点。这样无需遍历整个树。

这样的搜索策略性优化还有很多，会在之后的搜索专题里详细阐述。

剪枝

有时候，这个搜索过程中会出现很多无意义的分支，我们可以在搜索阶段就删除，这种策略我们称之为剪枝，这是一个十分形象的比喻，也非常的tricky，在我打NOIP的时候就特别喜欢用这种办法优化搜索。千万不要觉得这些小技巧不入流，有时候这些小技巧远胜于那些高级算法；我个人觉得，我打NOIP时本来就技不如人，就是靠这种技巧性拿的奖。这种技巧性需要在实战中多总结经验，多尝试，才能运用熟练，出类拔萃。

当然，技巧性另一方面也说明了，这个技能确实不是一个非常具有普适性的作用的技能。例如上面的算法题就用不了这个技巧。具体问题具体分析，只能是这样。

算法入门番外1：排序的那些事

发表于 2023-12-07 分类于 算法入门 ， 番外 本文字数： 1.4k

前段时间整理排序那块的知识，感觉放哪都不合适，但是又很重要，所以干脆开个番外，把排序里一部分内容重新整理一下。

比较排序

从实用的角度来看，建议直接用sort代替这里的所有排序方法，它们面对sort毫无优势。但从学习的角度看，这些算法里的思想，还是相当值得学习的。

归并排序

什么是归并？归并就是合并（merge），这个排序算法的设计是基于合并两个有序数组的算法思想来的。将两个有序数组合并为一个有序数组是简单的，因此，对一个有序数组的排列问题就转换为对这个有序数组的前半段数组和后半段数组进行排序的问题，而这恰好是原问题的两个子问题，符合分治的规则。代码参考这篇博客，简洁明了。

这种排序方法时间复杂度为\(O(n\log n)\)，空间复杂度由于要多开一个辅助数组，所以是\(O(n)\)。

小tips：这种排序方式可以用来计算逆序对（对于\(a[i]\)和\(a[j]\)，若有\(i < j\) 且 \(a[i] > a[j]\)，则称\((i,j)\)是一对逆序对）的个数，貌似只有这个\(O(nlogn)\)的方法。

堆排序

虽然stl十分强大，但是咱们还是要好好学习理论。堆排序的核心是维护一个二叉堆，这是一个特殊的二叉树，若父节点取值均大于子节点，则被称为最大堆，若父节点取值均小于子节点，则被称为最小堆。这里涉及二叉树的储存和读取，会在后面的章节中学到。

该排序从叶子节点开始，慢慢向上维护二叉堆性质，从而达到建立二叉堆的目的。在建立二叉堆之后，每次取出堆顶元素（表现为与倒数第i个数交换），调整二叉堆，不断循环这两步，即可完成对数组的排序。

由于调整堆（heapify）的过程最差时间复杂度就是\(O(\log n)\)，因此算法整体的时间复杂度就是\(O(n\log n)\)。空间复杂度是\(O(1)\)。虽然stl优先队列不讲道理

快速排序

快速排序的思想仍然是分治的思想，只不过这次的分治要保证的是：数列被划分为相对大小关系明确的两部分（一部分中的任意一个元素大于另一部分中的任意一个元素）。

那如何解决这个问题呢？双指针可以解决。假设想让左部小于右部，一个指针p指向数列首部，一个指向尾部p。q先启动往前扫描，找一个小于base的值，覆盖p的值，然后p启动，找一个大于base的值，覆盖q的值。当两指针重合时，用base覆盖那个值，这个问题就都得到解决了。

线性排序算法

编程里有种思想叫做“时间换空间”。算法快了，那么，代价是什么呢？空间。这里的排序算法更加容易理解，也更好写，因此简单带过。

计数排序

我印象中它还被叫做统计排序，就是统计数字出现的次数。oi-wiki上三句话就讲明白了：

  1. 计算每个数出现了几次；
  2. 求出每个数出现次数的 前缀和；
  3. 利用出现次数的前缀和，从右至左计算每个数的排名。

显然，其时间复杂度为\(O(n+w)\)，空间复杂度为\(O(n+w)\)

桶排序

桶排序按下列步骤进行：

  1. 设置一个定量的数组当作空桶；
  2. 遍历序列，并将元素一个个放到对应的桶中；
  3. 对每个不是空的桶进行排序；
  4. 从不是空的桶里把元素再放回原来的序列中。

这个领会个思想就行了。时间和空间都和选的排序算法有关系。

基数排序

这种排序和桶排序很像。只不过，它是对数的每一位进行拆解，根据那一位来决定把数放到哪个桶里，然后排下一位，直到最后所有数字都被排好。

没有看到特别优雅的代码，也没有看到特别好的解析，也不常用，就这样吧。

算法入门2-5：枚举和搜索基础

发表于 2023-12-06 更新于 2023-12-09 分类于 算法入门 ， 基础算法和基本思想 本文字数： 1k

枚举和搜索其实存在一定的关联，二者的核心思想都是尝试所有可能答案。在基础学习中，这两块也经常一同出现。

枚举

枚举，就是把所有可能的答案都尝试一遍，判断是否符合题目条件。在这个过程中，通常空间复杂度不成问题，而时间复杂度可能在两个部分产生：
1. 枚举所有可能的答案
2. 判断是否符合条件

针对后者优化的空间不大，许多题目都是围绕前者进行的优化。不过由于此类方法过于暴力，常常都是结合数学或者其他知识，枚举仅作为一个方法工具。

搜索

尝试所有可能答案说来容易，但是要做到其实并不容易。和枚举一样，很多情况下尝试所有答案意味着爆炸的时间复杂度（指数爆炸都算比较低的）。因此，搜索方法出现就是为了更快的获取答案。

DFS

一个简单的例子，输出一个数组的全排列（不要调库）。如果采用简单的for循环枚举，由于数组有多少元素就有多少层循环，这种代码无法完成。这时候就要用到递归搜索。由于这种搜索和树的遍历存在某种一致性，这种递归搜索也被称为深度优先搜索（DFS, Depth First Search）。递归可以参考算法入门2-4：递归和分治。

递归我们已经了解了，那么递归搜索又是什么呢？严格的说，应该是用递归的方法枚举答案。每层递归会（以同样的规则）确定答案的一部分，并且可以确定在某一层（边界）必然可以完全确定答案并进行判断。

例如全排列，可以把数组的每一个元素都看作答案的一部分，每层递归确定一个元素，最终可以得到一个全排列。

这两个概念其实并不适合并列在一起，因为二者并不是同一级别的关系。递归可以说是一种思考方式，而分治更像一种解决问题的具体方法。但之所以我把这二者放在一起，因为二者有时候确实联系相当紧密。这章被我命名为“基础算法和基本思想”也正是因为如此。这很像“哲学理论”和“由理论延申出来的观点思想”二者的关系。

递归

许多新手对它的认识仅限于：

但是其实这涉及到一个很重要的思考方式，递归式的思考，我想借用子结构的概念来表达这样的意思。

其实很难明确给子结构下一个定义，按照人类通俗的理解方式，问题的子结构满足以下两个条件：

  1. 问题本身的答案可以自己的子结构的答案通过简单的推导得到，而子结构的答案也可以由二阶子结构以同样的方式推导得到
  2. 存在某一阶子结构可以直接得到答案

  2. 根据经验，递归还要注意层数不能太深，不然会出现栈溢出的风险。（同样问题递归写法空间消耗大于非递归）

既然递归是一种思考方式，所谓的题也就显得比较单薄了，而且这样的思考方式还会推导出其他方法。应该说，单独的例题是没办法体现出这种子结构思想的美妙之处的。这个思想十分重要，建议所有人全文理解。

分治

在递归中，我们已经提到了子结构及其重要性。分治即是一种利用子结构解题的方法。所谓分治，就是分而治之。具体分为如下三步：

  1. 分解原问题为结构相同的子问题
  2. 分解到容易求解的边界，返回到某个值
  3. 合并子问题的解

二分法本来也算一个很基础的算法，但是其实题目难度会略大于前缀和和差分，因此放到了这里。

升序数组中查找一个数

这是二分法最基本的用法。在无序数组中，要查找是否有一个数，必须逐个进行查询，但升序数组不同。我们把要查询的数称为key，现在先假设这个数在mid的位置，如果mid位置的数比key小，那么key必然在这个数后面；反之，如果mid位置的数比key大，那么key必然在这个数前面。这就是二分法查询的思路。

从代码上来看，二分实质上是维护了一个答案区间\([start, end]\)，通过不断缩小答案区间来判断答案所在的位置。

二分答案

上面说的是基础中的基础，下面说的的才是真正实用的内容。

问题：要求符合某个条件的（在某个范围里）最大（或者最小）的数，

其实”顺序“这个概念是可以推广的。具体而言，如果在一个范围里，答案存在单向优化的情况，就可以称这个答案区间是”有序“的。

“单向优化”是什么？这说的是答案本身的性质。它表现为，如果一个答案可以被排除，那么它前面或者后面所有的答案都能被排除，这才是判断是否可用二分的法则。

这句话也可以表述为，检验一个值不为key时，可以准确地判定key所在的方向（更大或者更小）。这个判断准则，甚至可以随二分过程而改变，往往容易让这题变得没那么显而易见，这也就是为什么这种题有时候是很难的。

还有一些基本的题型，大矩阵第k小（对应下面第四题），最好有这样的解题思路。

没有题总是显得这些话语单薄了些，来吧，这里的题都相对难一点，不是那么容易想到的：

  1. 483. 最小好进制 @ Leetcode

  3. 4. 寻找两个正序数组的中位数 @ Leetcode
  4. 668. 乘法表中第k小的数 @ Leetcode
  5. 878. 第 N 个神奇数字 @ Leetcode

最大值最小化

我们常常会遇到诸如“选定某个值，使得最大值最小”的问题。这是一个经典的二分答案题型，二分最小最大值，然后去判断是否符合答案，符合就强化假设（假设更小的最小最大值），否则弱化假设。

如果上面的题刷明白了，其实这个很简单，看到就能秒了。但是有时候，这个问题并不那么明显，需要做一点转化，有难度的题一般会在这里挖坑，这类题并不多。

经典题型：

  1. 410. 分割数组的最大值 @ Leetcode
  2. 363. 矩形区域不超过 K 的最大数值和 @ Leetcode

总结

二分算法本身很简单，但衍生出来的题思考都带有一些难度，有时候还会结合动态规划等其他场景做一些更加复杂的结合，需要大家仔细思考。

算法入门2-2：前缀和和差分

发表于 2023-11-28 更新于 2023-12-08 分类于 算法入门 ， 基础算法和基本思想 本文字数： 1.1k

学习本节内容可先了解前缀和，二维前缀和和差分，重点掌握其思想，进阶技巧会在后续内容详细阐述。

前缀和

前缀和，顾名思义，对于一个数组\(\{a_i\}\)，它的第\(i\)个前缀和表现为：
\[ pre\_sum_{i} = \sum_{j=0}^ia_j \]
它的作用在于\(O(1)\)时间复杂度内求出数组从\(a_i\)到\(a_j\)的代数和。即：
\[ sum_{ij} = pre\_sum_{j}-pre\_sum_{i-1} \]

计算前缀和时，可用递推的方式计算：
\[ pre\_sum_i = pre\_sum_{i-1} + a_i \]

看完这些就可以完成一些简单的前缀和题目了。

二维前缀和

二维前缀和和一维前缀和类似，对于一个二维数组\(\{a_{ij}\}\)，它的前缀和表现为：
\[ pre\_sum_{(r,c)} = \sum_{i=0}^r\sum_{j=0}^ca_{(i,j)} \]
它的作用在于\(O(1)\)时间复杂度内求出任意连续子数组（可以简单理解为，用一个方框框住这个数组的一部分，这一部分就是子数组）的代数和。假设要求的子数组第一个（即左上方元素）位置为\((r_1,c_1)\)，最后一个元素为\((r_2,c_2)\)，则有：
\[ sum_{(r_1,c_1)\to(r_2,c_2)} = pre\_sum_{(r_2,c_2)}-pre\_sum_{(r_1-1,c_2)}-pre\_sum_{(r_2,c_1-1)}+pre\_sum_{(r_1-1,c_1-1)} \]
这个式子是基于容斥原理的，偷了张图，看着这张图，上面的公式应该好理解：

计算前缀和时，借助上面的公式很容易就得到：
\[ pre\_sum_{(r,c)} = a_{(r,c)}+pre\_sum_{(r-1,c)}+pre\_sum_{(r,c-1)}-pre\_sum_{(r-1,c-1)} \]

后缀和

既然可以算前缀，那么后缀和的求法也是一样的。虽然前缀和能表达后缀和，但是这个概念的出现就是为了帮助思考问题的。后缀和通常和前缀和一起出现解决问题。

前缀积

这个用的很少，因为乘法多了很容易指数爆炸。知道即可。

树上前缀和（了解）

树的前缀和指的是从一个节点到根节点的权值之和，可以是点权也可以是边权。它的出现是为了能在树上任何一条路径上进行点权和边权的离线统计。例如：
\[ sum_{(i,j)} = pre\_sum_i + pre\_sum_j - pre\_sum_{lca(i,j)} \]
其中，LCA是最近公共祖先，图论部分会详细介绍其求法。

差分

差分概念如下：
\[ \Delta_i = a_i-a_{i-1} \]
这不就是前缀和公式吗有什么意义？格局小了。它的作用在于\(O(1)\)时间复杂度内对一个区域内的数做加减运算。准确的说应该是，\(O(n)\)执行n次对一片区域做线性运算的操作。对于一个数组\(a_i\)而言，差分的前缀和实际就是一对逆运算，因此，一次区域的加减运算可以转化为对差分数组的\(O(1)\)操作。

具体而言，若需要对\(a_i\)到\(a_j\)之间的所有元素加上\(x\)，只需要维护一个差分数组，初始化为0。每次对区域\(a_i, a_{i+1}, a_{i+2}, ... a_j\)操作时，让\(\Delta_{i} = x, \Delta_{j+1} = -x\)，在完成\(n\)次操作后，用前缀和累加差分数组，即可获得\(n\)次操作后每个元素需要改变的值。

同理可以结合高维问题和后缀和衍生出各种各样的差分，根据前缀和的概念和推导，相信这些结论都可以顺利推导出来，这里不再赘述。

tips:这个要求离线操作，即修改必须在查询之前，否则这个操作没有任何意义。在线操作需要用到线段树。

树上差分（了解）

总结来说，差分和前缀和可以解决的无非就是区域离线统计和操作。而树上差分则是将这个概念在数据结构中的进一步扩展。具体而言，树上差分解决的是对树上任何一条路径上的点权和边权的离线统计和操作。图论部分会详细介绍其用途。

总结

差分和前缀和可以解决批量的区域离线统计和操作问题。

极限就是邻域内趋于某个值，多元函数也不例外。只不过这里自变量的“趋于”用的是范数定义的。（以二元函数为例）其含义是：无论定义域中的点以任何形式趋近，都趋于一个常数。因此，证明二重极限不存在就可以：

  1. 两条路径（）极限不相等
  2. 某条路径极限不存在
