Java梦的起点6：多线程

发表于 2024-01-26 更新于 2024-02-04 分类于 后端 ， Java 阅读次数： 本文字数： 1.6k

进程是执行程序的一次执行过程。进程中可以包括若干个线程。线程是CPU调度和执行的单位。线程的运行由调度器安排（不是立即启动），无法认为干预，并且会产生额外的开销。

main函数就是主线程，垃圾回收的线程叫gc线程。

synchronized关键字可以保证被它修饰的方法或者代码块在任意时刻只能有一个线程执行，它解决的是多个线程之间访问资源的同步性。

这说的是synchronized关键字的作用。方法和代码块只能由一个线程执行。而且，前提是对同一个对象来说。

死锁

死锁（deadlock），当两个以上的运算单元，双方都在等待对方停止执行，以获取系统资源，但是没有一方提前退出时，就称为死锁。

这种情况的出现的前提肯定是两个线程都是同时持有多个对象的锁。嵌套同步块（或者是同步函数里有其他同步块）的情况是允许的，但是务必确认没有死锁的情况。尽可能的，使用完一个对象，然后释放锁，再去锁另一个对象，就可以解决死锁的问题。

生产者消费者问题

这其实是一个线程间通信的问题。在这个问题中有三个对象，生产者，产品和消费者。生产者和消费者共享一个资源，而且生产者和消费者互为条件。这就涉及线程间通信问题。

常用的通信方法有：wait()（释放锁，并让线程进入等待状态）和notify()（随机唤醒一个线程），notifyAll()（唤醒所有线程）。这些方法都只能在同步方法和同步代码块中使用，否则会抛出异常。

解决这个问题的几种方法：
1. 管程法：建立缓存区，生产者往缓冲区里放东西，消费者从里面取
2. 信号灯法：生产者通过改变信号灯，通知消费者来取东西

其实我感觉，信号灯法就是缓存为1时候的管程法，感觉用处也不是很大。

线程池

频繁创建和销毁会使用特别多的资源，在并发情况下对性能影响很大。线程池的思路就是提取创建好多个线程，使用时直接获取，使用完放回池中。有点共享单车的意思。

除了降低消耗和提升性能，线程池还有便于管理的特点。
