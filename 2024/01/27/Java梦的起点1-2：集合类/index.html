Java梦的起点1.2：集合类

发表于 2024-01-27 更新于 2024-02-06 阅读次数： 本文字数： 1.6k

集合其实就是一个容器。我习惯用c++写算法了，因此我更愿意称其为“Java里的STL”。但是我感觉Java里集合类的讲究还是比较深的。虽然说是调接口，但还是要了解其中的原理。接下来我们慢慢分析。这个最好是看完Java梦的起点3：多态、继承和抽象类后，然后再学习这一节。

由于相关资料更新迭代速度相对还是比较快的，我们还是以比较新的JDK 17为例了解这一节。参考文章：【Java 数据结构】集合类 (精华篇)

Java里常用的类其实都在两个体系里，一个是Collection体系，一个是Map体系。这两个体系都是用具体的类继承接口来实现的。为什么要用接口呢？因为接口是可以继承的，而且接口也可以实现多个接口，这样，具体的数据结构类就可以直接继承接口，而我们学习时也就只需要学习接口就可以了。

Collection，其实翻译就是集合。但此“集合”非彼数学意义上的集合(Set)，而是一种“容器”(Container)，代表着一种数据结构。所以集合类的定义在java.util包下。下图是集合体系的继承树：

Set

Set接口是Collection接口的子类，最大的特点就是不允许重复的元素。所以它有以下常用方法：

Set是一个无序集合。即，无论是添加还是删除，Set都不会按照对象进入顺序进行保存对象。

有两种常用的Set实现：HashSet和TreeSet。

HashSet

HashSet是一个由哈希表实现的无序集合，本质上是一个HashMap。它允许null元素的出现。当然，我们知道哈希表实现的的contains方法和add方法的时间复杂度都是\(O(1)\)。

TreeSet

TreeSet不仅实现了Set接口，还实现了SortedSet接口，它是一个有序的集合（注意区分这里的有序和Set本身的无序，这是两个意义上的概念），排序规则是从小到大。它不允许出现null元素。它的实现是红黑树。

Queue
